# 文档库 Dropzone 批量上传设计

## 目标

扩展文档管理页顶部 Dropzone，使用户能够一次拖入或选择多个文件，并复用现有批量上传弹窗完成确认与顺序上传。

## 范围

本次仅修改 `frontend/src/pages/DocumentList.tsx`：

- Dropzone 隐藏文件输入支持多选。
- Dropzone 接收多个文件后，将完整文件列表传入上传弹窗。
- 上传弹窗继续统一选择文档类型。
- 多文件标题分别使用去除扩展名后的文件名。
- 单文件仍允许编辑标题。
- 合法与非法文件同时进入弹窗；非法文件标红、展示原因并在提交时跳过。
- 合法文件继续按顺序调用现有单文件上传接口。

不修改：

- `POST /documents/upload` 接口及请求结构。
- 后端文件解析、入库、向量化和索引流程。
- 数据库、Docker、认证和依赖配置。

## 数据流

```text
拖入/选择 FileList
  -> 转换为 File[]
  -> 创建上传队列并校验扩展名
  -> 打开 UploadFileModal
  -> 展示全部文件及校验状态
  -> 顺序上传合法文件
  -> 非法文件标记为跳过
  -> 汇总成功、失败、跳过数量
  -> 刷新文档列表
```

## 组件调整

### DocumentList

- 将 Dropzone 的预选状态由单个 `File | null` 改为 `File[]`。
- `openUploadModal` 接收可选文件数组。
- `handleDropzoneFiles` 不再拒绝多文件，也不在进入弹窗前过滤非法文件。
- 隐藏文件输入增加 `multiple`。
- Dropzone 文案调整为支持批量拖入。

### UploadFileModal

- 将 `initialFile` 参数改为 `initialFiles`。
- 初始化及重置时使用完整文件数组创建 `UploadQueueItem[]`。
- 沿用现有格式校验、状态展示、顺序上传和结果汇总逻辑。
- 当队列仅有一个合法文件时展示标题输入；多文件时自动使用各自文件名。

## 错误处理

- 不支持格式：保留在文件列表中，标记为无效并展示支持格式，提交时跳过。
- 单个合法文件上传失败：标记失败并继续后续文件。
- 全部文件无效：禁止有效上传并提示没有可上传文件。
- 上传进行中：禁止关闭弹窗或重复提交。

## 验收标准

1. 拖入两个及以上合法文件后，弹窗展示所有文件并可顺序上传。
2. Dropzone 点击选择文件时也允许多选，行为与拖入一致。
3. 合法和非法格式混合拖入时，非法文件标红，合法文件仍可上传。
4. 单文件拖入仍可编辑标题。
5. 多文件上传完成后停留在文档列表，并展示成功、失败和跳过数量。
6. 支持格式保持 `.pdf/.docx/.txt/.md/.csv/.xlsx`。
7. 前端执行 `npm run build` 通过。

## 风险与控制

- 文件数量较多时仍采用顺序上传，避免同时触发多个解析、向量化和索引任务。
- 本次不设置文件数量或总大小上限，沿用后端单文件限制和错误反馈。
- 不抽取新组件，避免扩大当前已有未提交改动的影响范围。
